What is the Git workflow?

Git workflow - this is the principle of working with git in a team.

What is it for?

A set of commands and practices for managing code changes using the Git version control system. This is necessary to organize the development process, minimize conflicts and simplify the integration of changes.

How does it work?

Git workflow works with using branches. Developers create separate branches for each feature or task. Then they give it to testers and programmers to check quality, presence of bugs, etc. After successfully passing the review and approval, the developers merge the new branch into the main branch.

What is the differense between git rebase and git merge?

Merge creates a new commit that merges changes from both branches, preserving the entire history and creating a "branched" structure, while rebase moves commits from one branch to the top of another, creating a more linear history without additional merge commits. Using merge introduces an additional merge commit, while rebase makes the history cleaner and more consistent. With merge, conflicts are resolved once during the merge, but with rebase, conflicts can occur on every commit, which requires more effort to resolve. Merge is typically used to integrate changes into the master branch, and rebase is often used to prepare local changes before merging to keep the history linear.

Merge saves the complete history of changes, and rebase makes it more accurate and understandable.

What is the differense between git fetch and git pull?

Git fetch fetches changes from a remote repository, updating local references to remote branches, but does not make changes to your current branch. This allows you to see what changes have been made in the remote repository without directly merging them into your work. You can then review these changes and decide how to integrate.Git pull is a combination of two commands: git fetch and git merge. When you do a git pull, it first downloads changes from a remote repository and then automatically merges them into your current branch. This is a more automated process that allows you to quickly update your local branch, but can lead to conflicts if changes on the remote branch are not compatible with your local changes.

Git fetch gives you control over the update process, while git pull allows you to quickly integrate changes, but requires more attention to potential conflicts.
